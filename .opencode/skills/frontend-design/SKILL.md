# Frontend Design — Premium Australian Organic Interfaces

Use this skill when creating or improving frontend interfaces for apps that need to feel premium, distinctive, usable, handcrafted, and visually memorable.

The goal is to produce interfaces that look like they were created by a high-end independent design studio: polished, art-directed, tactile, minimal, strange enough to remember, and practical enough to use.

Default style:

- Australian-informed
- organic and flowing
- minimal but handcrafted
- premium rather than cute
- practical rather than decorative
- tactile rather than glossy
- strange enough to feel personal
- refined enough to feel expensive

Avoid generic AI-generated frontend design: purple gradients, floating glass cards, default SaaS dashboards, endless rounded rectangles, meaningless blobs, sterile white panels, and typography that looks like it came free with a microwave.

The final standard is not “nice”.

The final standard is: **premium, useful, distinctive, and finished**.

# When To Use This Skill

Use this skill whenever the task involves:

- Designing a new web app UI
- Improving an existing frontend
- Creating a landing page
- Styling React components
- Designing a dashboard
- Building a creative tool
- Designing a portfolio page
- Creating a design system
- Improving visual polish
- Making an app feel less generic
- Turning a functional prototype into something presentable
- Adding responsive, accessible UI
- Creating HTML/CSS layouts
- Refining UX copy and interaction states

Use this skill especially when the app is one of the user’s personal projects, portfolio projects, creative tools, music tools, LEGO-related apps, visual generators, dashboards, counters, or strange little web toys.

# Core Design Philosophy

Before writing code, define a clear visual direction.

Ask:

1. What is the app for?
2. Who is using it?
3. What emotion should the interface create?
4. What should feel distinctly Australian about it?
5. What is the one visual detail someone will remember?
6. What should be restrained?
7. What should feel handmade?
8. What makes this feel premium rather than merely pleasant?

The design should be confident but not loud.

It should feel like it was made by a person with taste, not assembled from 14 Tailwind examples and a mild panic.

---

# Premium Design Standard

Every interface should look expensive.

“Expensive” does not mean glossy, over-animated, luxury-brand black-and-gold, or painfully minimal in the way a $900 chair is minimal because it hates your spine.

It means the UI should feel:

- Art-directed
- Confident
- Carefully spaced
- Typographically mature
- Visually restrained but memorable
- Built from a coherent system
- Slightly unexpected
- Finished to a level that suggests a high-end design studio was involved

The interface should not look like a default component library with a nicer colour palette.

It should feel composed.

A user should be able to glance at the product and think:

> Someone cared about this.

# Expensive UI Rules

## 1. Spacing Discipline

Use generous, intentional whitespace.

Do not cram.

Do not let elements float randomly.

Every gap should feel chosen.

Bad:

- Equal padding everywhere
- Dense cards
- Cramped forms
- Layout sections with no breathing room
- Components packed together because the data technically fits

Good:

- Large calm margins
- Deliberate content grouping
- Strong vertical rhythm
- Enough negative space for the interface to feel designed, not merely filled
- Clear separation between primary, secondary, and supporting content

Premium UI often comes from what is left out.

## 2. Typography As The Main Design Material

Premium UI usually has better typography than decoration.

Use:

- A distinctive display face
- A calm, readable body face
- Deliberate type scale
- Strong line-height decisions
- Beautiful heading composition
- Editorial rhythm
- Intentional contrast between labels, body, data, and calls to action

Avoid:

- Default system font everywhere
- Weak headings
- Generic SaaS typography
- Inconsistent text sizes
- Uppercase labels sprayed everywhere like seasoning
- Monospace used as a personality substitute

Typography should carry personality before icons, illustrations, or gradients are added.

## 3. Fewer, Better Visual Ideas

Do not add ten decorative treatments.

Pick one or two strong visual ideas and execute them well.

Examples:

- Irregular paper-cut panels
- Soft music-staff inspired dividers
- Premium botanical geometry
- Abstract map-like shapes
- Sculptural clay-like controls
- Hand-drawn annotation marks
- Beautifully framed app previews
- Tactile counters and gauges
- Layered cards that feel like printed objects

The interface should not feel decorated.

It should feel art-directed.

## 4. Materiality

Use subtle visual material cues so the interface feels tactile.

Preferred:

- Paper grain
- Soft canvas texture
- Ink-like borders
- Ceramic matte surfaces
- Sun-washed cards
- Layered print-like panels
- Soft physical shadows
- Slightly uneven edges
- Pressed button states
- Muted texture overlays

Avoid:

- Plastic gradients
- Glassmorphism
- Glossy tech surfaces
- Fake 3D
- Heavy skeuomorphic texture
- Anything that looks like a crypto dashboard from 2021

Premium does not need shine.

Often, matte looks more expensive.

## 5. Controlled Imperfection

Handcrafted details should be precise, not sloppy.

Use:

- Slight irregularity
- Off-grid accents
- Organic curves
- Asymmetrical hero layouts
- Small drawn marks
- Gentle shape variation
- Slightly offset decorative borders

Avoid:

- Random wobble
- Messy borders
- Inconsistent alignment
- Childish hand-drawn styling
- Quirky elements that make the app harder to use

The imperfection should feel designed by someone with taste, not caused by a CSS accident.

## 6. High-End Colour Restraint

Use a restrained palette with one memorable accent.

Good premium palettes:

- Warm paper, charcoal, eucalyptus, clay
- Cream, storm blue, ochre, ink
- Bone, olive, rust, muted lavender
- Sand, faded cobalt, bark, wattle
- Off-white, bottle green, terracotta, soft black
- Linen, ink, dusty blue, banksia yellow

Avoid:

- Too many colours
- Raw Tailwind defaults
- Bright generic gradients
- Pure white everywhere
- Corporate blue
- Purple AI sludge
- Uncontrolled rainbow palettes

Colours should feel mixed, not picked from a dropdown.

## 7. Custom-Feeling Components

Even simple components should feel designed.

Buttons should have:

- Satisfying shape
- Clear hierarchy
- Tactile hover state
- Tactile press state
- Strong label clarity
- Proper disabled state

Cards should have:

- Considered proportions
- Subtle layering
- Organic but controlled radii
- Meaningful hierarchy
- Premium shadows or borders

Forms should have:

- Beautiful field spacing
- Calm validation
- Clear affordance
- Useful helper text
- Accessible labels
- Error states that help rather than scold

Navigation should feel:

- Simple
- Confident
- Slightly branded
- Not like dashboard boilerplate

## 8. Strong Hero Composition

For landing pages and creative tools, the first screen matters.

The hero should include:

- One memorable visual motif
- Excellent heading typography
- Clear product explanation
- One primary action
- A crafted supporting detail
- Enough negative space to feel confident

Avoid:

- Generic centred headline
- Three feature cards immediately below
- Abstract blob background with no meaning
- Huge empty hero with vague copy
- “AI-powered” as the only idea

The first screen should set the visual world.

## 9. Micro-Interactions With Taste

Use small, polished interactions.

Examples:

- Buttons depress slightly
- Panels drift into place
- Hover states feel tactile
- Active elements get a soft ink-like highlight
- Generated outputs animate in gently
- Selected items feel physically chosen
- Counters tick subtly
- Charts ease into view
- Playback states have a clear pulse

Avoid:

- Springy toy animations
- Excessive motion
- Constant floating blobs
- Hover effects on everything
- Slow animations that delay work
- Motion that makes the product feel like a cereal ad

Premium motion is usually quiet.

## 10. No Default AI/App-Builder Smell

Reject anything that looks like:

- Default shadcn
- Default Tailwind
- Default Vercel template
- Generic SaaS dashboard
- Purple gradient hero
- Glass cards
- “Beautiful modern interface” prompt output
- Lucide icons everywhere with no styling
- Identical rounded cards in a grid
- Placeholder copy pretending to be strategy
- Bland design-system minimalism
- Dribbble nonsense with no product logic

If it looks like it came from a tutorial, redesign it.

# Shape Language

Use organic and flowing shapes as a recurring design motif.

Preferred:

- Blob-like panels with restrained irregularity
- Softly warped cards
- Uneven pill shapes
- Layered paper cut-out forms
- Wavy section dividers
- Gentle curves
- Hand-drawn separators
- Offset backgrounds
- Slightly imperfect outlines
- Fluid containers that guide the eye
- Rounded forms that feel natural rather than algorithmic

Avoid:

- Perfectly generic rounded cards everywhere
- Overusing circles
- Random decorative blobs with no relationship to content
- Shape clutter
- Corporate abstract gradient blob backgrounds
- Anything that looks like a startup’s 2021 pitch deck had a baby with a lavender diffuser

Organic shapes should support hierarchy and movement.

They are not confetti.

# Layout Principles

Prefer layouts that feel composed rather than templated.

Use:

- Generous spacing
- Clear hierarchy
- Soft asymmetry
- Offset cards
- Layered content blocks
- Editorial-style sections
- Bento layouts only when softened with organic edges or handmade details
- Large hero areas with one strong visual idea
- Clear task-first flows for apps and tools
- Calm grouping for dense information
- Strong empty space around important controls

Avoid:

- Centered hero + three cards + testimonial strip unless specifically required
- Dense dashboards unless the product genuinely needs density
- Symmetrical layouts that feel like placeholder wireframes
- Over-polished component libraries with no personality
- Data tables that feel like spreadsheet cosplay
- Random decorative sidebars

For functional apps, clarity wins.

The design can be charming, but users still need to complete the task without playing “Where’s Wally: Settings Edition”.

# Typography

Typography should feel characterful but readable.

Preferred approach:

- Use a distinctive display typeface for headings
- Use a highly readable body typeface
- Consider slightly warm, editorial, handmade, or humanist fonts
- Use type scale deliberately
- Let headings have personality
- Keep body text calm and useful
- Use numerals carefully for dashboards and counters
- Use tabular numbers where data alignment matters

Avoid:

- Inter as the default choice
- Roboto, Arial, or generic system fonts unless constrained
- Overly futuristic fonts
- Tech-bro monospace everywhere
- Font combinations that scream “template marketplace”
- Decorative fonts for long text
- Tiny body copy

Good typography should do some of the visual work, so the rest of the interface can stay restrained.

Suggested font directions:

- Editorial serif heading + humanist sans body
- Soft grotesk heading + readable sans body
- Display serif heading + compact data font for counters
- Friendly rounded display face + restrained sans body
- Lightly nostalgic heading face + practical interface font

Typography should be premium before it is quirky.

# Colour System

Use natural, Australian-informed palettes.

Default palette direction:

- Warm off-white or sand base
- Muted eucalyptus, sage, or olive
- Clay, rust, ochre, or terracotta accents
- Soft blue or storm-grey secondary tones
- Charcoal or ink for text
- One unexpected accent used sparingly

Colour should feel sun-warmed, tactile, and grounded.

Avoid:

- Purple-blue gradients
- Neon unless the app concept explicitly calls for it
- Pure white everywhere
- Corporate navy plus cyan
- Random rainbow palettes
- Colour contrast failures
- Overly pale text
- Low-contrast beige-on-beige mush

Use CSS variables for all major colours.

Example:

```css
:root {
  --colour-paper: #f5efe3;
  --colour-ink: #25231f;
  --colour-eucalypt: #7f9272;
  --colour-clay: #b76e4c;
  --colour-wattle: #d8a93f;
  --colour-storm: #6f8795;
  --colour-line: rgba(37, 35, 31, 0.16);
}
````

Always check accessibility contrast for text and interactive states.

* Hand-drawn line accents
* Light canvas or print-like texture

# Motion

Motion should feel gentle, physical, and useful.

Use:

* Small reveals
* Organic easing
* Slight drift or float on decorative shapes
* Soft hover lift
* Button press compression
* Staggered section entry
* Highlighting that feels like ink, paper, or light
* Subtle transitions between generated states
* Smooth but quick dashboard updates
* Gentle playback indicators for music tools

Avoid:

* Constant movement
* Bouncy startup animations
* Excessive parallax
* Motion that makes the product feel like a cereal ad
* Anything that blocks the user from getting on with it
* Animations longer than the user’s patience

Respect `prefers-reduced-motion`.

Example:

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

# Component Style

Components should feel simple, tactile, and intentional.

## Buttons

Buttons should have:

* Clear labels
* Rounded or softly irregular shapes
* Subtle pressed state
* Strong contrast
* Distinct primary and secondary styles
* Visible focus states
* Appropriate disabled state

Avoid:

* Generic glowing gradients
* Icon-only primary actions
* Low contrast
* Tiny hit areas
* Ambiguous labels

## Cards

Cards should have:

* Slightly organic corners
* Soft border or shadow
* Layered backgrounds
* Gentle offset or cut-paper feeling
* Strong internal hierarchy
* Enough padding to feel premium

Avoid:

* Endless identical cards
* Default card grids
* Weak visual grouping
* Borders that feel like wireframes

## Forms

Forms should have:

* Large enough fields to use comfortably
* Clear labels
* Clear validation
* Plain-language errors
* Helpful microcopy
* Calm visual states
* Mobile-friendly spacing

Avoid:

* Hostile red unless the issue is serious
* Placeholder-only labels
* Tiny checkboxes
* Mystery validation
* Forms that feel like tax software unless the app is literally tax software

## Navigation

Navigation should be:

* Simple
* Task-first
* Clear
* Friendly
* Plainly labelled
* Slightly branded

Avoid:

* Mystery meat navigation
* Too many nav items
* Overuse of icons
* Enterprise dashboard clutter

## Empty States

Empty states should be:

* Useful
* Warm
* Specific
* Slightly characterful
* Action-oriented

Avoid:

* Fake cheerfulness
* “No data” and nothing else
* Empty illustrations that explain nothing

Example:

> Nothing here yet. Connect a provider and Knut will start counting the damage.

## Loading States

Loading states should be:

* Calm
* Specific
* Branded where appropriate
* Fast-feeling
* Non-blocking where possible

Avoid:

* Generic spinners everywhere
* Skeleton screens that look like a hostage situation in grey rectangles
* Cute loading copy that gets annoying after the second use

For utility products like Knut Counter, data-heavy screens must remain premium.

Use:

* Clear grouping
* Calm cards
* Strong numerals
* Tabular numbers
* Soft charts
* Clear thresholds
* Warm alert states
* Compact but breathable layouts
* Friendly explanations of technical terms
* Human-readable usage summaries

Avoid:

* Dense spreadsheet layouts
* Raw tables as the main UI
* Aggressive red/green financial styling
* Too many tiny metrics
* Provider logos doing all the visual work
* Dashboard sameness

A premium dashboard should feel like a well-designed instrument panel, not a spreadsheet wearing perfume.

For counters:

* Make the number feel important
* Use clear units
* Use progress carefully
* Show reset periods plainly
* Make cost estimates visually distinct from usage
* Use thresholds and alerts sparingly
* Avoid panic-inducing visuals unless action is required

UI copy should be plain, useful, and lightly characterful.

Write like a helpful Australian product designer, not a brand strategist trapped in a lift.

Preferred:

* “Try another version”
* “Nothing here yet”
* “That didn’t work. Check the file and give it another go.”
* “Make it weird”
* “Save as PNG”
* “Preview melody”
* “Re-roll”
* “Connect provider”
* “Show usage”
* “Reset soon”
* “That looks expensive. Unfortunately, it is.”

Avoid:

* “Unlock”
* “Harness”
* “Supercharge”
* “Seamless”
* “Next-generation”
* “World-class”
* “Delight your users”
* “Transform your workflow”
* “Elevate your productivity”
* “Leverage powerful insights”

The product can have warmth and wit, but do not make every label a joke.

Users have things to do.

# Accessibility

Accessibility is not optional decoration.

Every interface must include:

* Semantic HTML
* Keyboard navigability
* Visible focus states
* Sufficient colour contrast
* Labels for form fields
* ARIA only where needed
* Reduced motion support
* Responsive layouts
* Touch targets suitable for mobile
* No information conveyed only by colour
* Proper heading structure
* Clear error messaging
* Accessible loading states
* Screen-reader friendly dynamic updates where needed

Organic design must not compromise usability.

A beautiful inaccessible interface is just a locked door with nice hinges.

# Responsive Design

Design mobile-first unless the product clearly requires desktop-first.

## Mobile

For mobile:

* Prioritise task completion
* Keep controls thumb-friendly
* Avoid tiny icon-only buttons
* Collapse complex layouts gracefully
* Preserve visual personality without cramming in decoration
* Use stacked sections
* Keep primary actions visible
* Avoid horizontal scrolling unless the content genuinely requires it

## Tablet

For tablet:

* Use two-column layouts selectively
* Let panels breathe
* Avoid awkward stretched mobile layouts
* Preserve touch-friendly controls

## Desktop

For desktop:

* Use space elegantly
* Avoid stretching everything to full width
* Create calm content zones
* Let organic shapes guide attention
* Use side panels only when useful
* Keep line lengths readable

Responsive design should feel deliberately composed at every size.

# Implementation Guidance

When coding:

* Use production-ready HTML, CSS, and JavaScript or the project’s existing framework
* Prefer CSS variables for theme tokens
* Keep components clean and reusable
* Avoid hardcoding magic values everywhere
* Use accessible semantic structure
* Keep animations performant
* Do not introduce unnecessary dependencies
* If using Tailwind, extend the theme so it does not look like default Tailwind
* If using React, keep components readable and composable
* Use real content where possible
* Design loading, empty, error, success, hover, focus, and disabled states
* Keep file structure understandable
* Do not bury the design in unnecessary abstraction

Match complexity to the project.

A tiny utility app does not need a design system the size of Bunnings.

# Preferred Visual Modes

Choose one clear visual mode per project.

Do not mix all modes at once. That way lies visual soup.

## 1. Sun-Bleached Utility

Best for:

* Practical tools
* Calculators
* Dashboards
* File utilities
* Productivity apps
* Simple generators

Characteristics:

* Warm paper background
* Charcoal text
* Eucalyptus accents
* Soft cards
* Hand-drawn dividers
* Minimal motion
* Very clear task flow
* Premium restraint

Use when the app needs to feel calm, useful, and trustworthy.

## 2. Coastal Workshop

Best for:

* Creative apps
* Music tools
* Image tools
* Maker projects
* Audio/visual experiments

Characteristics:

* Sand, faded blue, canvas, clay
* Layered panels
* Organic controls
* Slight paper/canvas texture
* Playful but usable
* Soft hand-built feeling

Use when the app should feel like a creative instrument or studio object.

## 3. Suburban Oddity

Best for:

* Weird portfolio apps
* Playful experiments
* LEGO tools
* Music toys
* Creative utilities
* Strange one-purpose products

Characteristics:

* Friendly wonky shapes
* Slightly nostalgic colour
* Handmade icons
* Offbeat layout
* Dry, plainspoken copy
* Carefully controlled weirdness

Use when the app should feel memorable and personal without becoming chaotic.

## 4. Native Garden Minimal

Best for:

* Polished products
* Serious tools
* Premium dashboards
* Apps needing restraint
* Content-heavy interfaces

Characteristics:

* Sage, cream, bark, ochre
* Very clean composition
* Subtle botanical influence without literal leaves everywhere
* Calm forms

Never default to:

* Inter + white background + purple gradient
* Generic glassmorphism
* Default shadcn look without modification
* Default Tailwind colours
* All cards, no composition
* Random pastel blob backgrounds
* Corporate SaaS dashboard sameness
* Startup landing page clichés
* Americanised growth-platform visual language
* Australiana clichés
* Overly polished interfaces that feel like nobody touched them
* Grey-on-white “minimalism”
* AI-generated decorative blobs
* Placeholder copy
* Lucide-icon soup
* Unstyled forms
* Cards inside cards inside cards
* Beige as a personality

Also avoid making everything beige.

Beige is not a personality.

# Output Expectations

When asked to design or build UI:

1. Identify the product purpose and user context.
2. Pick one visual mode.
3. Define the design rationale briefly.
4. Implement the interface with production-quality code.
5. Include responsive behaviour.
6. Include accessibility states.
7. Use real UI copy, not placeholder mush.
8. Make one memorable visual choice.
9. Keep the interface usable.
10. Avoid generic AI aesthetics.
11. Make it feel premium.
12. Check that it belongs near Sandwich Codes, Melodestiny, or Knut Counter visually.

If producing code, include:

* Semantic structure
* Theme tokens
* Responsive styles
* Hover states
* Focus states
* Empty states where relevant
* Error states where relevant
* Loading states where relevant
* Reduced-motion handling
* No unnecessary dependencies

If producing design guidance, include:

* Visual mode
* Colour direction
* Typography direction
* Layout direction
* Component principles
* Interaction notes
* Accessibility notes

# Quality Checklist

Before finishing, verify:

* Does the UI have a clear visual point of view?
* Would this look credible in a high-end design studio portfolio?
* Does it suit an Australian audience without clichés?
* Is it minimal but still handcrafted?
* Does it feel expensive?
* Are the organic shapes functional, not random garnish?
* Is the typography doing real work?
* Is the spacing calm and intentional?
* Are there fewer than three major decorative ideas?
* Is the colour palette grounded, mature, and accessible?
* Do components feel custom, not default?
* Is the handmade quality controlled rather than messy?
* Is the layout responsive?
* Are focus, hover, loading, empty, success, disabled, and error states designed?
* Does the app still feel practical?
* Does it belong near Sandwich Codes, Melodestiny, or Knut Counter visually?
* Would this be recognisable as something other than default AI output?

If the interface looks cute but cheap, refine it.

If it looks clean but generic, add art direction.

If it looks expensive but unusable, simplify it.

If it looks handmade but messy, tighten the system.

If it looks corporate, make it stranger.

If it looks strange but confusing, make it clearer.

The target is:

> Premium, useful, strange enough to remember.
